System Wiki

This is the living reference for how BaseballScorecard.org interprets MLB game data and renders it as a traditional scorecard. All diagrams on this page are rendered using the same code that draws the actual scorecards, so they always reflect the current version.

What is a Baseball Scorecard?

A baseball scorecard is a paper-and-pencil system used to record every event in a baseball game. Each batter gets a cell, and within that cell, the scorer marks what happened: the pitches thrown, the result of the at-bat, where runners moved on the bases, and how outs were recorded.

BaseballScorecard.org renders these scorecards digitally using live MLB data. Every pitch, every at-bat, every substitution is translated from the MLB Stats API (GUMBO feed) into the visual language of traditional scorekeeping. The system follows the Bob Carpenter scorebook layout with conventions from benjamincrom/baseball notation.

The Play Cell

The play cell is the fundamental unit. Each cell represents one batter's plate appearance in one inning.

  • Top-left: Out badge (numbered circle: 1, 2, or 3)
  • Top-right: Count (balls-strikes) and pitch sequence column
  • Center: Diamond (base paths) or large notation text
  • Below diamond: Scoring notation (e.g., G6-3, F8)
  • Bottom-left: RBI indicator diamonds
  • Bottom-right: Mini strike zone (when 10 or fewer pitches)
  • Left edge: PH sub line (dotted, before the at-bat)
  • Right edge: PR sub line (dotted, after the at-bat)
  • Bottom edge: Pitcher change line (dotted squares with stats)
Count calculation The count shows balls-strikes at the end of the at-bat. Balls: count B and * codes. Strikes: count C, S, F, W, T but cap at 2. Foul balls after 2 strikes do not increment.

Double play out rule Each out number appears exactly once per inning. The batter's cell shows only the batter's out. The other runner's out appears in their own cell. The same out number must never appear twice in the same inning.

Home Run Diamond

Live render: solid black diamond with white "HR" text.

Scored Runner (Hatch Lines)

Live render: 3 diagonal hatch lines clipped to the diamond boundary.

When a runner scores on a play that is not a home run, the diamond shows exactly 3 diagonal hatch lines. The lines are drawn at 45 degrees, evenly spaced (R x 0.5 apart), clipped to the diamond, and use the same stroke weight as base paths.

Hash Marks (Hits)

Short perpendicular lines on the HP-1B segment. Count matches the base reached: 1 for single, 2 for double, 3 for triple.

Out Markers

When a runner is retired, the path is drawn to the midpoint between bases (truncated) and a numbered circle marks where the out occurred. Only the batter's own out appears in the batter's cell.

Third Out Notch

A 45-degree diagonal line in the bottom-right corner of the cell where the 3rd out occurs, marking the end of the half-inning.

Pinch Hitter (PH)

Dotted line on the LEFT side of the play cell. The sub enters before the at-bat. The line is the "entry door" before the action.

Pinch Runner (PR)

Dotted line on the RIGHT side of the play cell. The sub enters after the at-bat. The line is the "exit door" after the batter's result.

Pitcher Changes

Horizontal dotted line at the bottom of the cell where the departing pitcher threw their last pitch. Includes accumulated stats: strikes / total pitches / strikeouts.

Substitution Rules

Rules
  1. PH = LEFT (before at-bat). PR = RIGHT (after at-bat).
  2. One player = one line. A player cannot be both PH and PR.
  3. Two lines on one cell only when two different players substitute.
  4. PH/PR types are never overwritten by defensive switch events.
  5. All sub lines use the same 5px dotted square pattern.
  6. In bat-around situations, PH/PR lines are suppressed.

Display Format

Each pitch renders as: speed (left) + type code (right). Example: 94 FF = 94 mph four-seam fastball. In-play pitches are bold. Color indicates ball/strike/in-play.

Mini Strike Zone

When 10 or fewer pitches, a mini zone appears at the bottom-right. Each pitch plotted at its actual location. Zone dimensions adjust per batter/pitcher height. Faded batter/pitcher silhouettes at 25% opacity.

ABS Challenge Badges

Purple square badge next to challenged pitches: W (won, call overturned) or L (lost, call stands). Text always white for contrast.

Lineup

  • Starters: Weight 800, default text color
  • Substitutes: Weight 600, sub color, horizontal dotted divider above
  • Player names link to Baseball Savant statcast pages
  • Below name: jersey number, season AVG, season OPS
  • Batting order encoding: slot x 100 + subIndex (e.g., 101 = slot 1, first player)

Indicator Icons

  • Out badge: Numbered circle (1/2/3) in top-left corner. Radius = SUB_CIRCLE_R x 1.45.
  • RBI diamond: Small filled diamond in bottom-left. One per RBI.
  • Sub letter: Sequential letters (A, B, C...) in lineup area, never in play cells.
  • All indicator icons share the same base size (SUB_CIRCLE_R) for visual consistency.

Design System

  • All borders: 1px solid var(--grid-line), square corners, no shadows
  • All table data: left-aligned (never centered)
  • Team abbreviations: never bold in tables
  • Never combine bold AND italic
  • All sub lines: 5px squares, 4px gaps, consistent everywhere
  • Section titles larger than column headers (hierarchy)
  • Reuse existing patterns, never introduce new styles when existing ones fit
